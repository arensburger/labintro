Lab communication & record keeping

The lab communicates largely via Element. You'll need me to create a space for you there and send you an invitation. Accessing the lab computers & Linux

This lab, and 99% of all bioinformatics labs use the Linux operating systems rather than Window or Mac OS. Partially it's for historical reasons, but it's also because many of the applications designed for bioinformatics run on Linux. There are a variety of different "flavors" of Linux out there, my preference is for ubuntu, but it's just a preference. Below are some details about the computers used in the lab and how to access them.

Computer Hardware

The lab contains several workstations all running ubuntu. You are welcome to get an account on those workstations and even access them from outside campus (see below). Talk to me if you want to set that up, otherwise you can come by anytime and work in the lab at a computer station. In addition to the lab workstations we also have access to the CPP computer cluster. This is a cluster of high powered computers, used for analysis that is more intense than what can be handled in the lab. If your project needs access to the cluster I can help you get to your account, and you can access it from anywhere. You can also install Linux on your computer, even alongside Windows. Accessing the Lab Computer

You can access the lab computer (named traviata) via the command line in Windows, MacOS, or a Linux based operating system. There are two things you'll need: 1) an account on the traviata computer, 2) the CPP VPN running if you're not on campus. I will provide you a user name and password for traviata, instructions for getting the VPN running on Windows or MacOS are on the CPP IT website. Once you have an account and the VPN is running you'll need to use a ssh client program to access traviata from home.

  • Windows

    Putty is a popular ssh client. Install the program (instructions here) and connect to the lab computer by filling in 134.71.67.54 for the host ip. Once connected you'll be asked to enter your user name and password that I will have provided you with.

  • Mac

    You can run an SSH command from the "Terminal" program on a Mac. Run the "Terminal" program and in the command line type "ssh 134.71.67.54". Once connected you'll be asked to enter your user name and password that I will have provided you with.

Once you've got the VPN running, start the program "Terminal" to get the command line interface, and type in "ssh 134.71.67.54" to log into the computer.

Leaning to use Command Lines

To communicate with the lab computer you'll need to use Bash shell command lines (don't worry too much about the terminology here, just know that the "Bash shell" is the program on the lab Linux computers that allows you to issue commands). Most of the power of this operating system comes from learning to issue command via the command line. Why? Imagine you want to count the number of times the human genome has the sequence AAATCGC. If you try to open a 3,000,000,000 letter long document in Microsoft Word you'll likely crash your computer. However, issuing the command line below will give you the answer quickly even on a typical laptop computer (provided it runs Linux):

grep -o "AAATCGC" insert genome file name here | wc -l

The structure of the command above will take a bit of work to understand, but hopefully you can see how useful command lines can be for bioinformatic work. Here's a basic tutorial on using a command lines in ubuntu. It's probably best to work through this tutorial once you have a way to access a Linux based machine.

Learning to write scripts

Script writting is essential for any advanced bioinformatic work. You do not need to known how to script to work in the lab, but if it's something you're thinking about then read on.

Which language should I learn?

This is nearly always the first question students ask when they think about learning to program. There’s no one answer, it depends on what you want to achieve. Think of programming languages as a set of tools in a toolbox; all toolboxes can solve basic problems, but some toolboxes are better suited to certain tasks than others. When it comes to bioinformatics, there are a number of programming languages that are commonly used. Three of the most commonly used are listed below along with their strengths and weaknesses.

  • R

    Strength: Very popular in many different fields (not just bioinformatic), many existing programs that can be combined to solve specific problem

    Weaknesses: Takes a statistical approach to problems which is not always well suited for very large bioinformatic data sets, may take the most time to “get the hang of” of all the languages listed here

  • Python

    Strength: Currently the most popular introductory programming language

    Weaknesses: This language becomes less useful when used for more computationally demanding problem.

  • Perl

    Strength: The best language for manipulation of text (e.g. genome sequences),

    Weaknesses: This is not currently a popular language, so not as much demand for it outside of niche specialties

There are many other languages (Java, C, etc.). Once you’ve mastered one programming language it becomes much easier to pick up other ones, most bioinformaticians end up using multiple languages to solve problem in different situations, even when they have a preferred scripting language.

I’ve picked a language, now what?

Now it’s time learn. There are as many ways to learn how to program, below are some resources but there are many, many others. Pick the one that work best for you.

  • Coursera, Programming for Everybody Four week online course that “aims to teach everyone the basics of programming computers using Python”
  • edX, Data Science: R Basics Eight week online course that will “help you master the basics of R”
  • O’Reilly, Beginning Perl for bioinformatics A book that introduces both Perl programming and bioinformatic skills together. An online version of this book is available through the CPP library.
