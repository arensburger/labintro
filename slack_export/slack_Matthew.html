A couple of things going on here. First, your grep command is not quite correctly written. You're asking grep to identify all the letters "A" in the file /mnt/data/traviata/GCA019973935.1/GCA019973935.1Tncv1.0_genomic.fna, so the ">" is unnecessary. It should be grep A /mnt/data/traviata/GCA_019973935.1/GCA_019973935.1_Tncv_1.0_genomic.fna Second, by default grep will return those lines that match your search parameters, but that's not exactly what you want in this case. For example, the line "AACAGGACCTTAAA" will be returned by your grep search because it contains the character "A", but there are 7 "A" characters on this line. A way around that is the -o argument of grep. The -o argument tell grep to only returns that part of the line that matches your search. Here's an illustration of this argument. In the command lines below I create a file called temp.fa that contains the line AACAGGACCTTAAA, then I use grep to search for the character "A" in that file but with the -o argument. I hope you can see how it works from the output. peter@faust:~/Desktop$ echo "AACAGGACCTTAAA" &gt; temp.fa peter@faust:~/Desktop$ grep -o A temp.fa A A A A A A A Rather than sending all those "A" to the terminal, I can redirect the output of the grep command to a file called "out.txt" like this peter@faust:~/Desktop$ grep -o A temp.fa &gt; out.txt If you modify this command to your genome file, you'll get an output file with one line for every "A" character in the genome. The next thing will be to count how many lines that output file has. You can do that with the bash command "wc" with the "-l" argument, look it up here. As far as your Network error, I suspect that putty is having a hard time keeping up with all the lines you're printing to the screen. You can avoid that problem by sending your output to files rather than the screen, using the ">" as I did above.

Let me know if you have any questions, there's a bunch of stuff to process here. Good luck.
