Well done. Next, as this is a new genome the lab is working on, I'll need you to gather some statistics about it. Specifically:

  1. How many pieces it's divided into
  2. The percentage of A, C, T, G, and N (N=nucleotide unknown) nucleotide
  3. The size distribution of the genome sections (i.e. are there lots of small genome segments, or just a few big ones). You're going to use command lines to figure each of these things out. For now, we'll start with the number of pieces the genome is divided into. The genome file that ends in .fna is formatted in the FASTA format. This means that every segment of the genome starts with a line like this ">blah blah" (you can use the command more to see this for yourself). Therefore, what you need to do is to count the number of lines that start with the ">" symbol and you'll have the number segments. To count the number of such lines, you can use the grep command on the command line. You can read up about how to use the grep command, but basically it will look something like this on the command line grep -c "&gt;" &lt;path to the .fna file name&gt; where: ‚Ä¢ -c tells the grep command to return the number of matches to your pattern ‚Ä¢ ">" is the pattern to search for (the quote marks are necessary because the > symbol has special meaning otherwise on the command line) <path to the .fna file name> is the full path name to the .fna file

This may all be a lot to digest, so take your time looking into it and get back to me when you've figured out the number of segments in the genome, or need some help. Good luck.

FASTA format
In bioinformatics and biochemistry, the FASTA format is a text-based format for representing either nucleotide sequences or amino acid (protein) sequences, in which nucleotides or amino acids are represented using single-letter codes. The format allows for sequence names and comments to precede the sequences. It originated from the FASTA software package, but has now become a near universal standard in the field of bioinformatics.The simplicity of FASTA format makes it easy to manipulate and parse sequences using text-processing tools and scripting languages.

<p>Well done. Next, as this is a new genome the lab is working on, I'll need you to gather some statistics about it. Specifically:</p> <ol><li>How many pieces it's divided into</li><li>The percentage of A, C, T, G, and N (N=nucleotide unknown) nucleotide</li><li>The size distribution of the genome sections (i.e. are there lots of small genome segments, or just a few big ones). You're going to use command lines to figure each of these things out. For now, we'll start with the number of pieces the genome is divided into. The genome file that ends in .fna is formatted in the <a href="https://en.wikipedia.org/wiki/FASTA_format">FASTA format</a>. This means that every segment of the genome starts with a line like this "&gt;blah blah" (you can use the command <a href="https://www.geeksforgeeks.org/more-command-in-linux-with-examples/">more</a> to see this for yourself). Therefore, what you need to do is to count the number of lines that start with the "&gt;" symbol and you'll have the number segments. To count the number of such lines, you can use the <a href="https://www.geeksforgeeks.org/grep-command-in-unixlinux/#">grep</a> command on the command line. You can read up about how to use the grep command, but basically it will look something like this on the command line <code>grep -c "&amp;gt;" &amp;lt;path to the .fna file name&amp;gt;</code> where: ‚Ä¢ -c tells the grep command to return the number of matches to your pattern ‚Ä¢ "&gt;" is the pattern to search for (the quote marks are necessary because the &gt; symbol has special meaning otherwise on the command line) &lt;path to the .fna file name&gt; is the full path name to the .fna file</li> </ol> <p>This may all be a lot to digest, so take your time looking into it and get back to me when you've figured out the number of segments in the genome, or need some help. Good luck.</p>
